Answer 1: When you initialize a git repository, you are creating a 
.git extension directory. Need to do it to track versions of ordinary 
folders.

Answer 2: Staging area is the intermediate stage between 
working directory in the local machine and the 
repository. It allows to track which files have yet to be 
updated and/or look at which commits need to go into the repo.

Answer 3: We can use the staging area to make stage several changes 
that belong to a single logical grouping of changes that can be 
committed.

Answer 4: Branches help in grouping future changes from the current 
master version. They help create different flavors of the same program
